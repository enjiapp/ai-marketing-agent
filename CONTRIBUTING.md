## What We're NOT Looking For

- Code changes to the MCP server (it's proprietary and hosted)
- Changes to the API or tool definitions
- Security-sensitive information

## How to Contribute

### For Small Changes (Typos, Small Fixes)

1. Click the "Edit" button on the file in GitHub
2. Make your changes
3. Submit a pull request
4. We'll review and merge if appropriate

### For Larger Contributions

1. **Open an Issue First** - Describe what you want to add/change

### For New Examples

If you want to contribute example prompts or workflows:

1. Add them to the appropriate file in `/examples/`
2. Include:
   - Clear description of the use case
   - Sample prompt
   - Expected output/behavior
   - Industry or business type
3. Make sure examples are:
   - Realistic and practical
   - Well-explained
   - Family-friendly
   - Respectful

## Documentation Style Guide

- **Be Clear**: Use simple language
- **Be Specific**: Include concrete examples
- **Be Helpful**: Think about what users need to know
- **Be Concise**: Get to the point
- **Use Headings**: Make content scannable
- **Link Generously**: Reference related docs and enji.co pages


## Questions?

- **General Questions**: Open a GitHub Issue with the "question" label
- **Enji Product Questions**: Contact contact@enji.co
- **Documentation Questions**: Tag your issue with "documentation"

## Code of Conduct

Be kind, respectful, and constructive. We're all here to help small business owners succeed with their marketing.

Harassment, spam, or inappropriate content will result in removal from the project.

Contributors who make significant improvements to the documentation will be acknowledged in the README.
